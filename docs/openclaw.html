¿Qué es OpenClaw?

OpenClaw es un asistente personal de IA que ejecutas en tus propios dispositivos. Responde en los canales que ya usas (WhatsApp, Telegram, Slack, Discord, Signal, iMessage, y muchos más), puede hablar y escuchar en macOS/iOS/Android, y renderiza un Canvas interactivo que controlas.

Característica Descripción

Local-first

Todo se ejecuta en tu máquina. No hay nube de terceros.

Multi-canal

20+ canales de mensajería integrados.

Multi-modelo

OpenAI, Anthropic, Mistral, Google, y más.

Multi-agente

Routing de canales a agentes aislados.

Skills

Habilidades extensibles via ClawHub.

Apps companion

macOS, iOS, Android opcionales.

Comando gateway — Control del Gateway

Definición

El Gateway es el plano de control central de OpenClaw. Es un servidor WebSocket que coordina sesiones, canales, herramientas y eventos. Todos los componentes (CLI, apps, agents) se conectan al Gateway.

Sintaxis

openclaw gateway [options] [command]

Opciones Principales

Opción Tipo Default Descripción

--port <port>

number

18789

Puerto para el WebSocket del gateway

--bind <mode>

string

loopback

Modo de bind: loopback | lan | tailnet | auto

--auth <mode>

string

none

Modo de autenticación: none | token | password

--password <password>

string

-

Password para modo auth=password

--token <token>

string

-

Token para modo auth=token

--tailscale <mode>

string

off

Exposición Tailscale: off | serve | funnel

--verbose

flag

false

Logging detallado

--force

flag

false

Mata proceso existente en el puerto

Subcomandos

Comando Descripción

run

Ejecuta el gateway en foreground

start

Inicia el servicio (systemd/launchd)

stop

Detiene el servicio

restart

Reinicia el servicio

status

Estado del servicio + probe

health

Health check del gateway

discover

Descubre gateways via Bonjour

probe

Reachability + discovery + health

install

Instala el servicio

uninstall

Desinstala el servicio

Ejemplos Detallados

# Iniciar gateway básico
openclaw gateway run

# Iniciar en puerto específico con verbose
openclaw gateway run --port 19001 --verbose

# Iniciar con autenticación por password
openclaw gateway run --auth password --password mi_password

# Iniciar con acceso LAN
openclaw gateway run --bind lan

# Iniciar con Tailscale (tailnet-only)
openclaw gateway run --tailscale serve

# Iniciar con Tailscale público
openclaw gateway run --tailscale funnel --auth password --password mi_password

# Forzar reinicio
openclaw gateway run --force

# Ver estado
openclaw gateway status

# Ver health
openclaw gateway health

# Instalar como servicio
openclaw gateway install
openclaw gateway start

Modos de Bind

Modo Descripción Caso de uso

loopback

Solo localhost (127.0.0.1)

Desarrollo local, máximo seguridad

lan

Todas las interfaces de LAN

Acceso desde otros dispositivos en la red

tailnet

Solo vía Tailscale

Acceso remoto seguro sin exponer públicamente

auto

Detección automática

Selecciona según configuración

Comando agent — Ejecutar Agente

Definición

El comando agent ejecuta un turno del agente via el Gateway. Puede enviar mensajes al agente, especificar nivel de thinking, cambiar agente, y entregar respuestas a canales.

Sintaxis

openclaw agent [options]

Opciones Principales

Opción Tipo Default Descripción

-m, --message <text>

string

-

Mensaje para el agente

--agent <id>

string

default

ID del agente (sobrescribe routing)

--thinking <level>

string

medium

Nivel de thinking: off | minimal | low | medium | high | xhigh

--channel <channel>

string

last

Canal de entrega: telegram | whatsapp | discord | slack | etc.

--deliver

flag

false

Envía respuesta al canal seleccionado

-t, --to <number>

string

-

Número E.164 del destinatario

--timeout <seconds>

number

600

Timeout en segundos

--local

flag

false

Ejecuta localmente (sin gateway)

--json

flag

false

Output como JSON

--verbose on|off

string

-

Persiste nivel verbose para sesión

Niveles de Thinking

Nivel

Uso Típico

Descripción

off

Respuestas directas

Sin thinking, respuestas inmediatas y rápidas

minimal

Tareas simples

Mínimo thinking, para consultas básicas

low

Consultas básicas

Thinking bajo, respuestas ágiles

medium

Uso general

Balance entre thinking y velocidad (default)

high

Análisis complejo

Thinking alto, para tareas que requieren análisis

xhigh

Máximo análisis

Thinking máximo, para análisis profundo

Ejemplos Detallados

# Mensaje simple
openclaw agent --message "Resume mi última sesión"

# Nueva sesión con número
openclaw agent --to +15555550123 --message "Hola, ¿cómo estás?"

# Agente específico
openclaw agent --agent ops --message "Analiza los logs"

# Thinking alto para análisis
openclaw agent --message "Revisa arquitectura" --thinking high

# Ejecutar localmente
openclaw agent --local --message "Test rápido"

# Entregar respuesta al canal
openclaw agent --to +15555550123 --message "Genera reporte" --deliver

# Cambiar canal de entrega
openclaw agent --agent ops --message "Reporte" --deliver --reply-channel slack --reply-to "#reports"

# Timeout personalizado
openclaw agent --message "Análisis largo" --timeout 1800

# Verbose persistido
openclaw agent --message "Debug" --verbose on

# Output JSON
openclaw agent --message "Estado" --json

# Mensaje simple

# Con media

# Como documento
openclaw message send --target +15555550123 --message "PDF" --media report.pdf --force-document

# Editar
openclaw message edit --channel telegram --target +15555550123 --message-id 123 --message "Editado"

# Eliminar
openclaw message delete --channel telegram --target +15555550123 --message-id 123

Comando channels — Gestión de Canales

Definición

El comando channels gestiona los canales de chat conectados. Permite añadir, configurar, conectar y desconectar canales.

Sintaxis

openclaw channels [options] [command]

Subcomandos

Comando Descripción

list

Lista canales configurados + auth profiles

status

Estado del canal del gateway

add

Añade o actualiza cuenta de canal

remove

Deshabilita o elimina cuenta

login

Enlaza cuenta (si soportado)

logout

Desconecta sesión (si soportado)

logs

Logs recientes del canal

capabilities

Capacidades del proveedor

resolve

Resuelve nombres a IDs

Ejemplos Detallados

# Listar canales
openclaw channels list

# Ver estado con probes
openclaw channels status --probe

# Añadir Telegram
openclaw channels add --channel telegram --token "123456:ABCDEF"

# Añadir Slack
openclaw channels add --channel slack --bot-token "xoxb-..." --app-token "xapp-..."

# Añadir Discord
openclaw channels add --channel discord --token "bot-token"

# Login a WhatsApp (QR)
openclaw channels login --channel whatsapp

# Login a Slack
openclaw channels login --channel slack

# Logout
openclaw channels logout --channel whatsapp

# Ver logs
openclaw channels logs --channel telegram

# Ver capacidades
openclaw channels capabilities --channel discord

# Resolver nombre
openclaw channels resolve --channel telegram --name @mychannel

Ejemplos Detallados

openclaw config unset agents.defaults.model

# Validar config
openclaw config validate

Comando cron — Trabajos Programados

Definición

El comando cron gestiona los trabajos programados via Gateway. Permite crear, editar, eliminar y ejecutar trabajos con expresiones cron.

Sintaxis

openclaw cron [options] [command]

Subcomandos

Comando Descripción

list

Lista trabajos cron

add

Añade trabajo cron

edit

Edita trabajo cron

enable

Habilita trabajo

disable

Deshabilita trabajo

rm

Elimina trabajo

run

Ejecuta trabajo ahora (debug)

runs

Historial de ejecuciones

status

Estado del scheduler

Ejemplos Detallados

# Listar trabajos
openclaw cron list

# Añadir trabajo diario a las 9am
openclaw cron add --name "daily-report" --cron "0 9 * * *" --message "Genera reporte diario"

# Añadir trabajo one-shot
openclaw cron add --name "reminder" --at "2026-04-22T10:00:00Z" --message "Recordatorio"

# Añadir con timezone
openclaw cron add --name "eu-meeting" --at "2026-04-22T15:00:00" --tz "Europe/Madrid" --message "Reunión EU"

# Editar trabajo
openclaw cron edit "daily-report" --cron "0 10 * * *"

# Habilitar/deshabilitar
openclaw cron enable "daily-report"
openclaw cron disable "daily-report"

# Eliminar trabajo
openclaw cron remove "daily-report"

# Ejecutar ahora (debug)
openclaw cron run "daily-report"

# Ver historial
openclaw cron runs

# Ver estado del scheduler
openclaw cron status

Creación de Agentes Personalizados

¿Qué es un Agente en OpenClaw?

Un agente en OpenClaw es una instancia aislada con su propia configuración, workspace, y comportamiento. Permite tener múltiples "personalidades" o "roles" que atienden diferentes canales o tareas.

Componente Descripción

ID del agente

Identificador único (ej: default, ops, support)

Configuración

Modelo, system prompt, temperatura, thinking level

Workspace

Directorio de trabajo con archivos de contexto

Skills

Habilidades específicas del agente

Canales

Routing de canales a este agente

Estructura de un Agente

Un agente se define mediante varios archivos en su workspace:

~/.openclaw/
├── agents/
│   ├── default/                    # Agente por defecto
│   │   ├── config.json             # Configuración del agente
│   │   ├── SOUL.md                 # Personalidad y tono
│   │   ├── AGENTS.md               # Reglas operativas
│   │   ├── TOOLS.md                # Herramientas disponibles
│   │   ├── MEMORY.md               # Memoria persistente
│   │   ├── USER.md                 # Información de usuarios
│   │   ├── IDENTITY.md             # Identidad del agente
│   │   └── skills/                 # Skills específicas
│   │
│   └── ops/                        # Agente de operaciones
│       ├── config.json
│       ├── SOUL.md
│       └── ...
│
└── config.json                     # Configuración global

Archivos de Definición del Agente

SOUL.md — Personalidad y Tono

El archivo SOUL.md define la personalidad, tono y estilo de comunicación del agente.

# Soul — Mi Agente

## Persona

You are **Mi Agente**, un asistente personal especializado en [dominio].
Tu estilo base es [nivel de formalidad], pero te adaptas al usuario.

## Tone & Language

- **Nivel 0**: Formal, sin slang, profesional
- **Nivel 1**: Neutral, internacional
- **Nivel 2**: Casual, expressions suaves
- **Nivel 3**: Latino medio, Spanglish ocasional
- **Nivel 4**: Latino intenso, slang frecuente
- **Nivel 5**: Máximo, full Spanglish

## Core Traits

- **Creativo**: Propone ideas rápidamente
- **Resuelve-rápido**: Va al grano
- **Cercano**: Trato amistoso
- **Profesional**: Cumple tareas con precisión
- **Discreto**: No expone información sensible

AGENTS.md — Reglas Operativas

El archivo AGENTS.md define las reglas de operación, prioridades y flujos de trabajo.

# Operational Rules — Mi Agente

## Priority Order

1. **Security first** - Nunca exponer datos sensibles
2. **User intent** - Ejecutar solicitudes claramente
3. **Quality** - Mantener calidad en las respuestas
4. **Tone** - Adaptar al nivel del usuario

## Step 0 — Sender Resolution (Mandatory)

### 0.1 Lookup sender identity
Extraer de metadata: `user_id`, `channel`, `display_name`

### 0.2 Auto-register new sender
Si no existe:
- Crear entrada en MEMORY.md
- Inicializar preferencias

### 0.3 Update activity
Actualizar `last_seen_at` para usuarios existentes

## Boundaries

- Never bypass access controls
- Never delete files you don't own
- Never expose private metadata

TOOLS.md — Herramientas Disponibles

El archivo TOOLS.md describe las herramientas y capacidades del agente.

# Tools — Mi Agente

## 1. User Registry

**Commands:**
- `/users` — Lista usuarios registrados
- `/me` — Muestra perfil propio
- `/bio <text>` — Actualiza biografía

## 2. Messaging

**Commands:**
- `/msg @user <text>` — Enviar mensaje directo
- `/send-file @user <file_id>` — Enviar archivo

## 3. Storage

**Commands:**
- `/space ls` — Listar archivos propios
- `/space rm <file_id>` — Eliminar archivo propio

## 4. Skills

Las skills se cargan desde `~/.agents/skills/`:

| Skill | Descripción
|-------|-------------
| `weather` | Clima y pronósticos
| `github` | Operaciones de GitHub

¿Qué es un Servidor MCP?

Un servidor MCP (Model Context Protocol) expone herramientas (tools), recursos (resources) y prompts a clientes de IA. OpenClaw puede conectarse a servidores MCP para extender sus capacidades.

Componente Descripción

Tools

Funciones ejecutables (acciones)

Resources

Datos accesibles (contexto)

Prompts

Plantillas predefinidas

Sampling

Solicitar generación al LLM

Elicitation

Solicitar input al usuario

Paso 1: Crear Directorio de Skills

mkdir -p ~/.agents/skills/mi-mcp-skill

# Mi MCP Skill

## Description
Integración con servidor MCP personalizado.

## Configuration


| Tool | Descripción

Crear un Servidor MCP Personalizado

Estructura del Proyecto

mkdir -p ~/mcp-servers/mi-servidor
cd ~/mcp-servers/mi-servidor
npm init -y
npm install @modelcontextprotocol/sdk

server.js — Servidor Básico

import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/transport/stdio.js';

const server = new Server({
  name: 'mi-servidor',
  version: '1.0.0'
}, {
  capabilities: {
    tools: {},
    resources: {}
  }
});

// Definir tools
server.setRequestHandler('tools/list', async () => {
  return {
    tools: [
      {
        name: 'mi_tool',
        description: 'Descripción de la tool',
        inputSchema: {
          type: 'object',
          properties: {
            parametro: { type: 'string' }
          },
          required: ['parametro']
        }
      }
    ]
  };
});

// Manejar ejecución
server.setRequestHandler('tools/call', async (request) => {
  const { name, arguments: args } = request.params;

  if (name === 'mi_tool') {
    return {
      content: [{
        type: 'text',
        text: `Resultado: ${args.parametro}`
      }]
    };
  }

  throw new Error(`Tool desconocida: ${name}`);
});

// Iniciar
const transport = new StdioServerTransport();
await server.connect(transport);

    args: { location: "Madrid" }
