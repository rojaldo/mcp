¿Qué es OpenClaw?

OpenClaw es un asistente personal de IA que ejecutas en tus propios dispositivos. Responde en los canales que ya usas (WhatsApp, Telegram, Slack, Discord, Signal, iMessage, y muchos más), puede hablar y escuchar en macOS/iOS/Android, y renderiza un Canvas interactivo que controlas.

Característica Descripción

Local-first

Todo se ejecuta en tu máquina. No hay nube de terceros.

Multi-canal

20+ canales de mensajería integrados.

Multi-modelo

OpenAI, Anthropic, Mistral, Google, y más.

Multi-agente

Routing de canales a agentes aislados.

Skills

Habilidades extensibles via ClawHub.

Apps companion

macOS, iOS, Android opcionales.

Comando gateway — Control del Gateway

Definición

El Gateway es el plano de control central de OpenClaw. Es un servidor WebSocket que coordina sesiones, canales, herramientas y eventos. Todos los componentes (CLI, apps, agents) se conectan al Gateway.

Sintaxis

openclaw gateway [options] [command]

Opciones Principales

Opción Tipo Default Descripción

--port <port>

number

18789

Puerto para el WebSocket del gateway

--bind <mode>

string

loopback

Modo de bind: loopback | lan | tailnet | auto

--auth <mode>

string

none

Modo de autenticación: none | token | password

--password <password>

string

-

Password para modo auth=password

--token <token>

string

-

Token para modo auth=token

--tailscale <mode>

string

off

Exposición Tailscale: off | serve | funnel

--verbose

flag

false

Logging detallado

--force

flag

false

Mata proceso existente en el puerto

Subcomandos

Comando Descripción

run

Ejecuta el gateway en foreground

start

Inicia el servicio (systemd/launchd)

stop

Detiene el servicio

restart

Reinicia el servicio

status

Estado del servicio + probe

health

Health check del gateway

discover

Descubre gateways via Bonjour

probe

Reachability + discovery + health

install

Instala el servicio

uninstall

Desinstala el servicio

Ejemplos Detallados

# Iniciar gateway básico
openclaw gateway run

# Iniciar en puerto específico con verbose
openclaw gateway run --port 19001 --verbose

# Iniciar con autenticación por password
openclaw gateway run --auth password --password mi_password

# Iniciar con acceso LAN
openclaw gateway run --bind lan

# Iniciar con Tailscale (tailnet-only)
openclaw gateway run --tailscale serve

# Iniciar con Tailscale público
openclaw gateway run --tailscale funnel --auth password --password mi_password

# Forzar reinicio
openclaw gateway run --force

# Ver estado
openclaw gateway status

# Ver health
openclaw gateway health

# Instalar como servicio
openclaw gateway install
openclaw gateway start

Modos de Bind

Modo Descripción Caso de uso

loopback

Solo localhost (127.0.0.1)

Desarrollo local, máximo seguridad

lan

Todas las interfaces de LAN

Acceso desde otros dispositivos en la red

tailnet

Solo vía Tailscale

Acceso remoto seguro sin exponer públicamente

auto

Detección automática

Selecciona según configuración

Comando agent — Ejecutar Agente

Definición

El comando agent ejecuta un turno del agente via el Gateway. Puede enviar mensajes al agente, especificar nivel de thinking, cambiar agente, y entregar respuestas a canales.

Sintaxis

openclaw agent [options]

Opciones Principales

Opción Tipo Default Descripción

-m, --message <text>

string

-

Mensaje para el agente

--agent <id>

string

default

ID del agente (sobrescribe routing)

--thinking <level>

string

medium

Nivel de thinking: off | minimal | low | medium | high | xhigh

--channel <channel>

string

last

Canal de entrega: telegram | whatsapp | discord | slack | etc.

--deliver

flag

false

Envía respuesta al canal seleccionado

-t, --to <number>

string

-

Número E.164 del destinatario

--timeout <seconds>

number

600

Timeout en segundos

--local

flag

false

Ejecuta localmente (sin gateway)

--json

flag

false

Output como JSON

--verbose on|off

string

-

Persiste nivel verbose para sesión

Niveles de Thinking

Nivel

Uso Típico

Descripción

off

Respuestas directas

Sin thinking, respuestas inmediatas y rápidas

minimal

Tareas simples

Mínimo thinking, para consultas básicas

low

Consultas básicas

Thinking bajo, respuestas ágiles

medium

Uso general

Balance entre thinking y velocidad (default)

high

Análisis complejo

Thinking alto, para tareas que requieren análisis

xhigh

Máximo análisis

Thinking máximo, para análisis profundo

Ejemplos Detallados

# Mensaje simple
openclaw agent --message "Resume mi última sesión"

# Nueva sesión con número
openclaw agent --to +15555550123 --message "Hola, ¿cómo estás?"

# Agente específico
openclaw agent --agent ops --message "Analiza los logs"

# Thinking alto para análisis
openclaw agent --message "Revisa arquitectura" --thinking high

# Ejecutar localmente
openclaw agent --local --message "Test rápido"

# Entregar respuesta al canal
openclaw agent --to +15555550123 --message "Genera reporte" --deliver

# Cambiar canal de entrega
openclaw agent --agent ops --message "Reporte" --deliver --reply-channel slack --reply-to "#reports"

# Timeout personalizado
openclaw agent --message "Análisis largo" --timeout 1800

# Verbose persistido
openclaw agent --message "Debug" --verbose on

# Output JSON
openclaw agent --message "Estado" --json

Sintaxis

openclaw message [options] [command]

Comando Descripción

send

Enviar un mensaje

Broadcast a múltiples targets

ban

Banear un miembro

kick

Kickear un miembro

thread

Acciones de thread

Opciones de message send

Opción Tipo Default Descripción

string

-

Destinatario (E.164, chat ID, @username, canal)

-m, --message <text>

string

-

Cuerpo del mensaje

--channel <channel>

string

-

Canal: telegram | whatsapp | discord | slack | etc.

--media <path-or-url>

path/url

-

Adjuntar media (image/audio/video/document)

--reply-to <id>

string

-

Responder a mensaje específico

--silent

flag

false

Enviar sin notificación (Telegram + Discord)

--force-document

flag

false

Enviar como documento (sin compresión)

--buttons <json>

json

-

Botones inline (Telegram)

--components <json>

json

-

Discord components

--thread-id <id>

string

-

Thread ID (Telegram forum)

--dry-run

flag

false

Ver payload sin enviar

Ejemplos Detallados

# Mensaje simple
openclaw message send --target +15555550123 --message "Hola!"

# Con media
openclaw message send --target +15555550123 --message "Foto" --media photo.jpg

# Como documento
openclaw message send --target +15555550123 --message "PDF" --media report.pdf --force-document

# Silenciosamente
openclaw message send --target +15555550123 --message "Notificación" --silent

# Reply
openclaw message send --target +15555550123 --message "Respuesta" --reply-to 12345

# En thread
openclaw message send --channel telegram --target -100123456 --message "En thread" --thread-id 789

# Con botones (Telegram)
openclaw message send --target +15555550123 --message "Elige:" --buttons '[["Opción 1"],["Opción 2"]]'

# Discord components
openclaw message send --channel discord --target 123456 --message "Select:" --components '{"type":1,"components":[{"type":3,"options":[{"label":"A","value":"a"}]}]}'

# Dry run
openclaw message send --target +15555550123 --message "Test" --dry-run

# Poll
openclaw message poll --channel discord --target channel:123 --poll-question "¿Pizza?" --poll-option Pizza --poll-option Sushi

# Reacción
openclaw message react --channel discord --target 123 --message-id 456 --emoji "✅"

# Leer mensajes
openclaw message read --channel telegram --target +15555550123

# Editar
openclaw message edit --channel telegram --target +15555550123 --message-id 123 --message "Editado"

# Eliminar
openclaw message delete --channel telegram --target +15555550123 --message-id 123

Comando channels — Gestión de Canales

Definición

El comando channels gestiona los canales de chat conectados. Permite añadir, configurar, conectar y desconectar canales.

Sintaxis

openclaw channels [options] [command]

Subcomandos

Comando Descripción

list

Lista canales configurados + auth profiles

status

Estado del canal del gateway

add

Añade o actualiza cuenta de canal

remove

Deshabilita o elimina cuenta

login

Enlaza cuenta (si soportado)

logout

Desconecta sesión (si soportado)

logs

Logs recientes del canal

capabilities

Capacidades del proveedor

resolve

Resuelve nombres a IDs

Ejemplos Detallados

# Listar canales
openclaw channels list

# Ver estado con probes
openclaw channels status --probe

# Añadir Telegram
openclaw channels add --channel telegram --token "123456:ABCDEF"

# Añadir Slack
openclaw channels add --channel slack --bot-token "xoxb-..." --app-token "xapp-..."

# Añadir Discord
openclaw channels add --channel discord --token "bot-token"

# Login a WhatsApp (QR)
openclaw channels login --channel whatsapp

# Login a Slack
openclaw channels login --channel slack

# Logout
openclaw channels logout --channel whatsapp

# Ver logs
openclaw channels logs --channel telegram

# Ver capacidades
openclaw channels capabilities --channel discord

# Resolver nombre
openclaw channels resolve --channel telegram --name @mychannel

Comando config — Configuración

Definición

El comando config gestiona la configuración de OpenClaw. Permite obtener, establecer y validar valores de configuración.

Sintaxis

openclaw config [options] [command]

Subcomandos

Comando Descripción

file

Imprime path del config activo

get

Obtiene valor por dot path

set

Establece valores por path

unset

Elimina valor por dot path

validate

Valida config contra schema

Dot Paths Comunes

Path Descripción

agent.model

Modelo por defecto

gateway.port

Puerto del gateway

gateway.bind

Modo de bind

gateway.auth.mode

Modo de autenticación

channels.telegram.botToken

Token de Telegram

channels.slack.botToken

Bot token de Slack

channels.discord.token

Token de Discord

agents.defaults.sandbox.mode

Modo sandbox

agents.defaults.workspace

Directorio workspace

Ejemplos Detallados

# Ver archivo de config
openclaw config file

# Obtener valor
openclaw config get agent.model

# Obtener valor anidado
openclaw config get channels.telegram.botToken

# Establecer valor simple
openclaw config set agent.model "openai/gpt-4o"

# Establecer con JSON estricto
openclaw config set gateway.port 19001 --strict-json

# Establecer con referencia a env var
openclaw config set channels.discord.token --ref-provider default --ref-source env --ref-id DISCORD_BOT_TOKEN

# Establecer secrets desde archivo
openclaw config set secrets.providers.vault --provider-source file --provider-path /etc/openclaw/secrets.json --provider-mode json

# Batch desde archivo
openclaw config set --batch-file ./config-set.batch.json --dry-run

# Eliminar valor
openclaw config unset agents.defaults.model

# Validar config
openclaw config validate

Comando models — Gestión de Modelos

Definición

El comando models gestiona los modelos de LLM disponibles. Permite listar, escanear, establecer modelos por defecto, y gestionar fallbacks.

Sintaxis

openclaw models [options] [command]

Subcomandos

Comando Descripción

list

Lista modelos configurados

scan

Escanea modelos OpenRouter gratuitos

status

Estado de modelos configurados

set

Establece modelo por defecto

set-image

Establece modelo de imágenes

aliases

Gestiona aliases de modelos

auth

Gestiona perfiles de autenticación

fallbacks

Gestiona lista de fallbacks

image-fallbacks

Gestiona fallbacks de imágenes

Ejemplos Detallados

# Listar modelos
openclaw models list

# Ver estado
openclaw models status

# Estado en JSON
openclaw models status --status-json

# Establecer modelo por defecto
openclaw models set "anthropic/claude-opus-4"

# Establecer modelo de imágenes
openclaw models set-image "openai/dall-e-3"

# Escanear OpenRouter
openclaw models scan

# Gestionar aliases
openclaw models aliases list
openclaw models aliases add "my-model" "anthropic/claude-opus-4"

# Gestionar auth
openclaw models auth list
openclaw models auth add --provider openai --api-key "sk-..."

# Gestionar fallbacks
openclaw models fallbacks list
openclaw models fallbacks add "openai/gpt-4o"
openclaw models fallbacks add "google/gemini-2.5-pro"

Comando cron — Trabajos Programados

Definición

El comando cron gestiona los trabajos programados via Gateway. Permite crear, editar, eliminar y ejecutar trabajos con expresiones cron.

Sintaxis

openclaw cron [options] [command]

Subcomandos

Comando Descripción

list

Lista trabajos cron

add

Añade trabajo cron

edit

Edita trabajo cron

enable

Habilita trabajo

disable

Deshabilita trabajo

rm

Elimina trabajo

run

Ejecuta trabajo ahora (debug)

runs

Historial de ejecuciones

status

Estado del scheduler

Ejemplos Detallados

# Listar trabajos
openclaw cron list

# Añadir trabajo diario a las 9am
openclaw cron add --name "daily-report" --cron "0 9 * * *" --message "Genera reporte diario"

# Añadir trabajo one-shot
openclaw cron add --name "reminder" --at "2026-04-22T10:00:00Z" --message "Recordatorio"

# Añadir con timezone
openclaw cron add --name "eu-meeting" --at "2026-04-22T15:00:00" --tz "Europe/Madrid" --message "Reunión EU"

# Editar trabajo
openclaw cron edit "daily-report" --cron "0 10 * * *"

# Habilitar/deshabilitar
openclaw cron enable "daily-report"
openclaw cron disable "daily-report"

# Eliminar trabajo
openclaw cron remove "daily-report"

# Ejecutar ahora (debug)
openclaw cron run "daily-report"

# Ver historial
openclaw cron runs

# Ver estado del scheduler
openclaw cron status

Flujos de Trabajo Comunes

Flujo: Instalación Inicial

# 1. Instalar
npm install -g openclaw@latest

# 2. Onboarding
openclaw onboard --install-daemon

# 3. Verificar estado
openclaw status

# 4. Configurar modelo
openclaw config set agent.model "anthropic/claude-opus-4"

# 5. Iniciar gateway
openclaw gateway start

# 6. Verificar canales
openclaw channels list

# 7. Diagnóstico final
openclaw doctor --fix

Flujo: Configurar Telegram

# 1. Añadir token
openclaw channels add --channel telegram --token "123456:ABCDEF"

# 2. Verificar estado
openclaw channels status --channel telegram

# 3. Configurar allowlist
openclaw config set channels.telegram.allowFrom --json '["123456789"]'

# 4. Probar envío
openclaw message send --channel telegram --target 123456789 --message "Test"

# 5. Ver logs
openclaw channels logs --channel telegram

Flujo: Acceso Remoto con Tailscale

# 1. Configurar Tailscale serve
openclaw config set gateway.tailscale.mode "serve"

# 2. Reiniciar gateway
openclaw gateway restart

# 3. Verificar
openclaw gateway status

# Para acceso público (funnel):
# openclaw config set gateway.tailscale.mode "funnel"
# openclaw config set gateway.auth.mode "password"
# openclaw config set gateway.auth.password "mi_password"
# openclaw gateway restart

Flujo: Auditoría de Seguridad

# 1. Auditoría básica
openclaw security audit

# 2. Auditoría profunda
openclaw security audit --deep

# 3. Reparar problemas
openclaw doctor --fix

# 4. Verificar config
openclaw config validate

# 5. Ver estado
openclaw status --deep
