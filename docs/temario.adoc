== OBJETIVOS DEL CURSO

Al finalizar el curso, el alumno será capaz de:

* Comprender la arquitectura y propósito del protocolo MCP.
* Implementar servidores MCP con herramientas y recursos personalizados.
* Conectar modelos de lenguaje con fuentes de datos externas.
* Desarrollar clientes que consuman servidores MCP.
* Aplicar buenas prácticas de seguridad y diseño.
* Integrar MCP con frameworks populares (Claude, OpenAI, LangChain).

== MÓDULO 1: INTRODUCCIÓN Y CONCEPTOS (3 horas)

=== 1.1. ¿Qué es MCP?

* Definición: Model Context Protocol
* Origen: Anthropic y la necesidad de estandarización
* Problema que resuelve: fragmentación de integraciones
* Comparación con alternativas: Function Calling, Plugins, Tool Use
* El ecosistema MCP: servidores, clientes, registry

=== 1.2. Arquitectura del protocolo

* Modelo cliente-servidor
* Transporte: stdio, HTTP/SSE, WebSocket
* Protocolo JSON-RPC 2.0
* Ciclo de vida de una conexión MCP
* Descubrimiento de capacidades

=== 1.3. Conceptos fundamentales

* *Resources*: datos y contenido accesible
* *Tools*: funciones ejecutables
* *Prompts*: plantillas de instrucciones
* *Sampling*: solicitudes de generación del modelo
* *Roots*: sistema de archivos accesible

=== 1.4. Instalación y configuración

* Instalación de SDK oficiales (Python, TypeScript)
* Configuración de Claude Desktop
* Configuración de Claude Code
* Inspector MCP: herramienta de depuración
* Estructura de un proyecto MCP

== MÓDULO 2: RECURSOS EN MCP (4 horas)

=== 2.1. Definición de Resources

* ¿Qué es un Resource en MCP?
* URIs y esquemas personalizados
* Tipos de contenido MIME
* Recursos estáticos vs dinámicos

=== 2.2. Implementación de Resources

* Decorador `@mcp.resource()` en Python
* Devolución de contenido: texto y binario
* Listado de recursos disponibles
* Metadatos y descripciones

=== 2.3. Recursos con plantillas

* URI Templates (RFC 6570)
* Parámetros dinámicos en recursos
* Ejemplo: acceso a base de datos por ID
* Validación de parámetros

=== 2.4. Recursos del sistema de archivos

* Acceso a archivos locales
* Lectura de directorios
* Filtros y permisos
* Seguridad: sandboxing y roots

=== 2.5. Casos prácticos

* Servidor de documentación local
* Acceso a archivos de configuración
* Recursos desde base de datos SQLite
* Recursos desde APIs externas

=== 3.3. Herramientas con efectos secundarios

=== 3.5. Casos prácticos

== MÓDULO 4: PROMPTS Y SAMPLING (4 horas)

=== 4.1. Definición de Prompts

* ¿Qué es un Prompt en MCP?
* Diferencia con prompts estáticos
* Argumentos y plantillas
* Prompts dinámicos

=== 4.2. Implementación de Prompts

* Decorador `@mcp.prompt()` en Python
* Argumentos con valores por defecto
* Construcción dinámica de mensajes
* Inclusión de recursos en prompts

=== 4.3. Sampling: el modelo genera

* ¿Qué es Sampling en MCP?
* Flujo de sampling: servidor solicita generación
* Configuración: modelo, temperatura, max_tokens
* Casos de uso: resumen, análisis, transformación

=== 4.4. Implementación de Sampling

* Decorador `@mcp.sample()` en Python
* Manejo de respuestas del modelo
* Sampling multimodal: texto e imágenes
* Restricciones de seguridad

=== 4.5. Casos prácticos

* Prompt de análisis de código
* Prompt de generación de documentación
* Sampling para clasificación automática
* Pipeline de procesamiento de documentos
